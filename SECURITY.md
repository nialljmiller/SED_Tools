# Security Policy

## Reporting a Vulnerability

If you discover a security vulnerability in SED_Tools, please report it privately — do not open a public GitHub issue.

**Options:**

- **Email:** niall.j.miller@gmail.com
- **GitHub:** Use the [private vulnerability reporting](https://docs.github.com/en/code-security/security-advisories/guidance-on-reporting-and-writing/privately-reporting-a-security-vulnerability) feature on this repository

Include as much detail as possible: a description of the vulnerability, steps to reproduce it, and any potential impact you have identified.

## Disclosure

Please give the maintainer reasonable time to investigate and address the issue before any public disclosure.

SED_Tools is provided as-is with no guarantees of security or fitness for any particular purpose. See the project license for full terms.
